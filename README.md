# MaxkorotZI

Защищенный P2P мессенджер (SPA) на Web-технологиях. Поддерживает сквозное шифрование (E2EE) и WebRTC для прямой передачи данных. Подходит для размещения на GitHub Pages и последующей мобильной сборки через Capacitor/Cordova.

## Возможности
- E2EE: ECDH (secp256k1) + AES-GCM 256
- Приватный ключ хранится только локально в браузере
- P2P канал через WebRTC DataChannel
- Сигналинг через публичный GunDB
- Mobile-first интерфейс

## Запуск локально
Откройте `index.html` в браузере. Для WebRTC рекомендуется HTTPS или `localhost`.

## Быстрый сценарий подключения
1. Оба пользователя вводят один и тот же `Room ID` и нажимают `Войти`.
2. Пользователь А копирует свой `ID` и `Публичный ключ`, отправляет Пользователю Б.
3. Пользователь Б вставляет `Peer ID` и `Публичный ключ`, нажимает `Создать соединение`.
4. После установления канала чат готов.

## Мобильная сборка (Capacitor)
1. Инициализируйте проект: `npx cap init MaxkorotZI com.maxkorotzi.app`
2. Добавьте платформу: `npx cap add android`
3. Укажите путь к статике: `npx cap copy`
4. Откройте Android Studio: `npx cap open android`

## Ограничения
- WebRTC требует STUN/TURN. Используется публичный STUN Google; для стабильной связи может понадобиться свой TURN.
- Публичные реле GunDB могут быть нестабильны; при необходимости замените на свои.
